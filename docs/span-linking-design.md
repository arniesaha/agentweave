# Span Linking Design for AgentWeave

## Problem
Currently, spans generated by sub-agents in AgentWeave appear as separate, disconnected traces in Grafana Tempo. This makes it difficult to visualize the full call hierarchy and understand the relationships between parent and child agents.

A typical example:
- Agent A spawns Agent B.
- The root span of Agent B should ideally appear as a child span in the trace generated by Agent A.
- However, without propagating the W3C Trace Context, Agent B starts a new trace, leading to disjointed visualization.

## Solution
Use W3C Trace Context propagation to link spans across agent boundaries. The `traceparent` and `tracestate` HTTP headers will propagate the trace context:

- **traceparent:** Contains information about the trace, such as the version, trace ID, parent ID, and trace flags.
- **tracestate:** Provides additional vendor-specific trace information.

When an agent spawns a sub-agent:
- The spawning agent injects the trace context into HTTP headers (or equivalent mechanism).
- The sub-agent extracts the context when starting its operations and links back to the parent span.

## Proposed API
### Injecting Trace Context
A helper method to inject the current span's context:
```python
from agentweave.propagation import inject_trace_context

headers = {}
inject_trace_context(headers)
# headers now contain 'traceparent' and optionally 'tracestate'
```

### Extracting Trace Context
A helper method to extract the parent context in a sub-agent:
```python
from agentweave.propagation import extract_trace_context

context = extract_trace_context(headers)
# Use this context when creating a new span in the sub-agent
```

### Utility for Traceparent
Retrieve the traceparent header value for debugging or logging:
```python
from agentweave.propagation import get_traceparent

print("Traceparent:", get_traceparent())
```
